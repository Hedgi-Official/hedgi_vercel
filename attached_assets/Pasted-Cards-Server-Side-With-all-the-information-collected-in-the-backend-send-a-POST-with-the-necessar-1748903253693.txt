Cards
Server-Side

With all the information collected in the backend , send a POST with the necessary attributes to the /v1/payments endpoint and execute the request or, if you prefer, send the information using our SDKs.

Important
Additionally, you must send the attribute X-Idempotency-Key. Filling it out is important to ensure the execution and re-execution of requests safely, without the risk of performing the same action more than once by mistake. To do so, update our SDK library or generate a V4 UUID and send it in the header of your calls.
          
const mercadopago = require('mercadopago');
import { MercadoPagoConfig, Payment } from '@src/index';

const client = new MercadoPagoConfig({ accessToken: '<ACCESS_TOKEN>', options: { timeout: 5000 } });

const payment = new Payment(client);

Brick's onSubmit callback contains all the data needed to create a payment, but if you wish, you can include additional details, which can make it easier for the buyer to recognize the purchase and increase the payment approval rate.

To do this, add relevant fields to the submitted object, which comes in the response from Brick's onSubmit callback .
