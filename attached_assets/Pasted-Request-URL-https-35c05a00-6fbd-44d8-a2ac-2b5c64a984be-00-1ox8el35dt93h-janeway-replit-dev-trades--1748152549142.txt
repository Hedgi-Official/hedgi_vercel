Whenever your client hits an endpoint that your Express server doesn’t recognize, it “falls through” into your SPA’s catch-all and serves index.html (hence the HTML you’re seeing). In your setup you have:

Express proxy routes under /api, e.g.

ts
Copy
Edit
app.post('/api/trades', …)
Your Dashboard was fetching /trades instead of /api/trades, so Express never saw it.

✔️ The one-line fix
In your Dashboard (and in CurrencySimulator) change:

ts
Copy
Edit
// was (wrong):
await fetch('/trades', { … })

// to (correct):
await fetch('/api/trades', { … })
Or, if you were using an API_BASE constant:

ts
Copy
Edit
// e.g.
// const API_BASE = '/api'

// …
await fetch(`${API_BASE}/trades`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload),
  credentials: 'include'
});
Why this matters
/api/trades → matches your Express proxy, which then forwards to Flask.

/trades → no match in Express, so your frontend server returns index.html instead of JSON.

Once you switch back to /api/trades, you should see a proper JSON response from your Flask backend (via the Express proxy), and your onSuccess handler will fire.
