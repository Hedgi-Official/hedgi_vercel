Standard rendering
Before rendering the Payment Brick, first perform the initialization steps shared between all Bricks. From there, see below the information you need to configure and render the Payment Brick.

Notice
For typings and specifications of Brick function parameters and responses, see the technical documentation .
Configure Brick
Create the Brick launch configuration.

          
const renderPaymentBrick = async (bricksBuilder) => {
 const settings = {
   initialization: {
     /*
      "amount" é o valor total a ser pago por todos os meios de pagamento com exceção da Conta Mercado Pago e Parcelamento sem cartão de crédito, que tem seu valor de processamento determinado no backend através do "preferenceId"
     */
     amount: 100,
     preferenceId: "<PREFERENCE_ID>",
   },
   customization: {
     paymentMethods: {
       ticket: "all",
       creditCard: "all",
       prepaidCard: "all",
       debitCard: "all",
       mercadoPago: "all",
     },
   },
   callbacks: {
     onReady: () => {
       /*
        Callback chamado quando o Brick estiver pronto.
        Aqui você pode ocultar loadings do seu site, por exemplo.
       */
     },
     onSubmit: ({ selectedPaymentMethod, formData }) => {
       // callback chamado ao clicar no botão de submissão dos dados
       return new Promise((resolve, reject) => {
         fetch("/process_payment", {
           method: "POST",
           headers: {
             "Content-Type": "application/json",
           },
           body: JSON.stringify(formData),
         })
           .then((response) => response.json())
           .then((response) => {
             // receber o resultado do pagamento
             resolve();
           })
           .catch((error) => {
             // lidar com a resposta de erro ao tentar criar o pagamento
             reject();
           });
       });
     },
     onError: (error) => {
       // callback chamado para todos os casos de erro do Brick
       console.error(error);
     },
   },
 };
 window.paymentBrickController = await bricksBuilder.create(
   "payment",
   "paymentBrick_container",
   settings
 );
};
renderPaymentBrick(bricksBuilder);

        
Attention
Whenever the user leaves the screen where a Brick is displayed, it is necessary to destroy the current instance with the command window.paymentBrickController.unmount(). When re-entering, a new instance must be generated.
To use the payment method ( paymentMethods) of the "mercadoPago" type, you must send a preference during Brick initialization, replacing the value <PREFERENCE_ID>with the ID of the preference created. Instructions for creating the preference are in the Enable payment with Mercado Pago section .

Enable payment with Mercado Pago
To use the payment method (paymentMethods) of the "mercadoPago" type, you must send a preference during Brick initialization, replacing the <PREFERENCE_ID> value with the ID of the preference created.

To create a preference in your backend, add the Mercado Pago SDK and the necessary credentials to your project to enable the use of preferences.

          
// SDK do Mercado Pago
const mercadopago = require ('mercadopago');
// Adicione as credenciais
mercadopago.configure({
  access_token: 'PROD_ACCESS_TOKEN'
});

        
Then set the preference according to your product or service.

The code examples below set the preference's purpose to wallet_purchase, but it is also possible to set it to onboarding_credits. Understand the difference between the two:

wallet_purchase : the user must log in when redirected to their Mercado Pago account.

onboarding_credits : after logging in, the user will see the credit payment option pre-selected in their Mercado Pago account.

        
// Cria um objeto de preferência
let preference = {
  // o "purpose": "wallet_purchase" permite apenas pagamentos logados
  // para permitir pagamentos como guest, você pode omitir essa propriedade
  "purpose": "wallet_purchase",
  "items": [
    {
      "id": "item-ID-1234",
      "title": "Meu produto",
      "quantity": 1,
      "unit_price": 75.76
    }
  ]
};
 
mercadopago.preferences.create(preference)
  .then(function (response) {
    // Este valor é o preferenceId que será enviado para o Brick na inicialização
    const preferenceId = response.body.id;
  }).catch(function (error) {
    console.log(error);
  });

      
Important
To learn more about how to configure it, go to the Preferences section.



Keep in mind that when a user chooses to make a payment using their Mercado Pago Account, they will be redirected to the Mercado Pago page to complete the payment. Therefore, it is necessary to configure it if you want to return to your website after completing the payment. For more information, visit the Redirect the buyer to your websiteback_urls section .
