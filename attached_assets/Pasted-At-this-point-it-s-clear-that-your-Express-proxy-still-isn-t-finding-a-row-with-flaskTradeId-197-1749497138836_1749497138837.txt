At this point it’s clear that your Express proxy still isn’t finding a row with flaskTradeId === 197, so it immediately returns your 404 “Trade not found”. Since you’ve already pared it down to a single findFirst({ where: eq(trades.flaskTradeId, flaskTradeId) }), the only thing left is to verify exactly what’s sitting in your DB and what your route is actually seeing.

1) Dump out all of your user’s trades in the proxy
Right at the top of your handler (before the lookup) add:

ts
Copy
Edit
// … inside your app.post('/api/trades/:flaskTradeId/close', …)

const userId       = req.user.id;
const flaskTradeId = Number(req.params.flaskTradeId);

console.log('[CLOSE DEBUG] param:', flaskTradeId, typeof flaskTradeId);

const allUserTrades = await db.query.trades.findMany({
  where: eq(trades.userId, userId)
});
console.log(
  '[CLOSE DEBUG] allUserTrades:',
  allUserTrades.map(t => ({
    dbId:         t.id,
    flaskTradeId: t.flaskTradeId,
    ticket:       t.ticket,
    symbol:       t.symbol,
    status:       t.status
  }))
);
Now hit the close-button in your UI. In your Express logs you’ll see exactly what values are in t.flaskTradeId. If none of them equals 197, that explains the 404.

3) Verify your create‐trade code really wrote the Flask ID
If your trades table’s flaskTradeId column was never populated at creation time, then every row will have flaskTradeId = null (or 0), and your lookup will always fail. In your create handler you should have done:

// after you get flaskBody.id back:
await db.query.trades.update({
  where: eq(trades.id, dbTrade.id),
  set:   { flaskTradeId: flaskBody.id }

If you skipped that, go back and run one of:

FROM trades

and make sure flask_trade_id matches the IDs you see in Flask itself.

4) (Optional) Return a debug payload to your client
If you want to see the same data in the browser instead of the terminal, you can also return it in the 404:
