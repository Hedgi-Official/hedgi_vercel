When the call comes from your website’s Express layer there’s an extra hop and a few little pitfalls that simply don’t exist when you run curl from your laptop. Nine times out of ten the failure is caused by one of the items below.

#	Where it goes wrong	What it looks like in the browser	How to verify / fix
1	Wrong URL assembled in Express (missing query-string, typo in the broker or symbol, hitting the HTTP not HTTPS port)**	Network tab: .../symbol_info 200 OK but body is {bid:0, ask:0…} or an error from Flask	console.log(fullUrl) right before you call fetch / axios; paste that URL into the browser’s address bar. If it gives 0/0 directly, the issue is not Express—double-check the query parameters.
2	Express proxies through Cloudflare, curl bypasses it	Cloudflare rule returns a cached stale response (0/0) or a 5 xx, while direct curl to the raw tunnel host works	From your EC2 box run: curl -H "Host: your.domain" http://127.0.0.1/symbol_info?... If that returns real prices, Cloudflare is interfering—disable cache or add a bypass rule for /symbol_info.
3	CORS pre-flight blocked, Express retries with mode:"no-cors" (giving an opaque response)	Response type “opaque”. Front-end tries to JSON-parse '' → error → you print {bid:0, ask:0} as fallback	Make sure Flask’s CORS(app, supports_credentials=True) is active and your Express proxy returns Access-Control-Allow-Origin: * (or the site’s origin).
4	Express forwards the request multiple times (retry loop), first one warms MT5, second one hits the 5 s timeout and you log “Failed to fetch rate…”	In Flask logs you see the same URL twice within milliseconds; one returns real bid/ask, the other zeros	Add console.log("called /quotes", Date.now()) in Express and print("Flask got", request.args) to see if the call is duplicated. Make sure your proxy middleware uses res.once("finish", …) not res.on("finish") when it retries.
5	Express strips the query string when it proxies (app.use("/quotes", proxy("...")) without proxyReqOptDecorator)	Flask route complains “No symbol provided” or falls back to default symbol that has 0/0	In Express, inspect req.originalUrl before proxying. Use something like: js proxyReqOptsDecorator: (opts, req) => { opts.path = `/symbol_info${req.url}`; return opts; }
6	IPv6 / dual-stack: browser hits the AAAA record, curl hits the A record; IPv6 is firewalled to the Flask port	Browser call hangs or fetches 0/0 after 5 s; curl (IPv4) is instant	In Developer Tools, check “Remote Address”. If it’s 2606:4700:…, disable AAAA at Cloudflare or open the port in the EC2 security group for IPv6.
7	Express uses keep-alive with many parallel tabs → bursts of 20–40 concurrent requests → hits the MT5 “reply-steal” race	Curl is fine (one request), React page returns zeros sporadically	Temporarily set agent: new http.Agent({keepAlive:false}) in the axios call: if zeros disappear you’re seeing the race. Then fix the race by adding per-request reply queues or UUID-tagged messages between Flask and the broker worker.
8	Your new EC2 runs Flask on port 5000, Express still points at http://host/symbol_info (port 80)	Browser gets a Cloudflare 522 or an Nginx 502	Hard-code the internal URL once: `const FLASK_URL = process.env.FLASK_URL
9	JSON parsing / numeric coercion in Express: you read `result.ask		0` before the promise resolves, log zeros
