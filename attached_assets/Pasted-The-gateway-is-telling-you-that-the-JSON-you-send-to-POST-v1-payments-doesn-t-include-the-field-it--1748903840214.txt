The gateway is telling you that the JSON you send to POST /v1/payments doesn’t include the field it needs to know which payment method (Visa, Master, Pix, etc.) the buyer chose.
For card (and most other) payments the five mandatory properties are

typescript
Copy
Edit
token              – one-time card/Pix token returned by the Brick
transaction_amount – number, same value you showed to the user
installments       – integer
payment_method_id  – e.g. "visa", "master", "pix"
payer.email        – string
``` :contentReference[oaicite:0]{index=0}

That `payment_method_id` must live at the **top level** of the JSON you send to Mercado Pago, and it must use snake_case exactly as above.  Two things usually make it disappear:

---

### 1  Not extracting it from the Brick’s callback

```ts
// React example with @mercadopago/sdk-react
const settings = {
  initialization: { amount },
  callbacks: {
    onSubmit: async (formData) => {
      /* formData already contains:
         formData.token
         formData.installments
         formData.paymentMethod.id   <- THIS is what you need
         formData.payer.email
         etc.                         */
      const payload = {
        token:               formData.token,
        transaction_amount:  Number(amount),
        installments:        Number(formData.installments),
        payment_method_id:   formData.paymentMethod.id,     // 👈
        issuer_id:           formData.issuer?.id ?? null,
        payer: { email: formData.payer.email }
      };

      const res = await fetch('/api/process_payment', {
        method : 'POST',
        headers: { 'Content-Type': 'application/json' },
        body   : JSON.stringify(payload)
      });
      return res.ok ? res.json() : Promise.reject(await res.json());
    }
  }
};
Verify in your browser console that payload really contains payment_method_id before the fetch.

2 Renaming it on the backend
If you do something like

js
Copy
Edit
const { paymentMethodId, ...rest } = req.body;
mpClient.post('/v1/payments', { paymentMethodId, ...rest });
Mercado Pago will not translate the camelCase name for you, so the field arrives null and the payment is rejected. Map it back:

js
Copy
Edit
const mpBody = {
  ...rest,
  payment_method_id: req.body.paymentMethodId   // snake_case here
};
Quick checklist
Check	Fix
- Log the body you POST to /v1/payments and confirm the field is there	console.log(JSON.stringify(body, null, 2))
- Stay in test credentials while debugging (production keys reject test tokens)	Use your TEST-*** keys everywhere
- Pass issuer_id for Amex/Arg-local cards (error 3012 otherwise)	issuer_id: formData.issuer.id
- Keep header X-Idempotency-Key to avoid duplicated charges while retrying	uuid.v4() each attempt

Once payment_method_id is present the "Invalid payment data: missing payment method" rejection disappears and you will see either status:"approved" or a more specific card-issuer error in the response.
