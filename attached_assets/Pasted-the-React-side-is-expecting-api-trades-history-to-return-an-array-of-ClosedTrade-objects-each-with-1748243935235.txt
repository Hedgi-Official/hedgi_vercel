the React side is expecting /api/trades/history to return an array of ClosedTrade objects, each with exactly these properties:

ts

interface ClosedTrade {
  id:        number;
  ticket:    string;
  symbol:    string;
  volume:    string;
  openTime:  string;
  closedAt:  string;
  status:    string;
}
but right now you’re handing it back the entire DB row (with userId, broker, metadata, etc.). That’s why your UI is seeing status: "open" and a closedAt: null—because you never projected away those unwanted fields or ran the Flask update logic in that handler.

How to fix /api/trades/history
In your Express GET /api/trades/history handler, build a fresh array of exactly the fields your UI needs. For example:

ts
Copy
Edit
app.get('/api/trades/history', async (req, res) => {
  // … auth, fetch allTrades, etc. …
  const historyTrades: ClosedTrade[] = [];

  for (const trade of allTrades) {
    if (!trade.flaskTradeId) continue;

    // 1) Fetch status from Flask
    const flaskRes = await fetch(`${FLASK}/trades/${trade.flaskTradeId}/status`);
    if (!flaskRes.ok) continue;
    const { status, closedAt: flaskClosedAt } = await flaskRes.json() as {
      status: string;
      closedAt?: string;
    };

    // 2) Only include completed trades
    if (!['closed','failed','executed','cancelled','completed']
          .includes(status.toLowerCase())) {
      continue;
    }

    // 3) Map into exactly the shape your UI expects
    historyTrades.push({
      id:       trade.id,
      ticket:   `FLASK-${trade.flaskTradeId}`,
      symbol:   trade.symbol,
      volume:   trade.volume.toString(),
      openTime: trade.createdAt.toISOString(),
      status,                                            // fresh from Flask
      closedAt: flaskClosedAt                            // ISO string
        ?? trade.updatedAt.toISOString(),                // fallback
    });
  }

  return res.json(historyTrades);
});
Why this works
You never return userId, broker, metadata, etc.

You always overwrite the old trade.status with the fresh status from Flask.

You always supply a non-null closedAt (either from Flask or your updatedAt).
