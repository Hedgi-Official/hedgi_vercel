Implement backend‑only trade tracking, open‑trades API, and cancel endpoint

Description:
Enhance our existing hedge‑creation and trade‑management backend so that:

On /hedge (POST):

Parse the MT5 response JSON and persist to our database one “Trade” record with these hidden fields:

ticket (MT5 order ID, from custom_order.ticket)

broker (e.g. "fbs", from the request context)

volume (from custom_order.volume or trade_details.volume)

symbol, open_time, duration_days, plus status=open.

Do not expose ticket or broker in the UI.

Open‑trades list (GET /api/trades/open):

Return only non‑sensitive fields for active trades:

json
Copy
[
  {
    "symbol": "USDMXN",
    "volume": "0.20 lots",
    "openTime": "2025‑05‑01T14:30:00Z"
  },
  …
]
Cancel trade (POST /api/trades/cancel):

Clients send { tradeOrderNumber: string } (the internal ID of our DB record).

Lookup that record’s ticket and broker.

Issue a close‑by‑magic request to the MT5 bridge:

jsonc
Copy
POST http://3.145.164.47/close_trade_by_magic
{
  "magic": <ticket>,
  "broker": "<broker>",
  "comment": "user‑initiated cancel"
}
On success, update trade status=cancelled and return { status: true, ... }.

On failure, return a 500 with { error: "Failed to cancel trade" }.

Closed‑trades history (GET /api/trades/closed):

Return a list of all trades with status = cancelled | closed_by_sl, with the same non‑sensitive shape plus one extra field closedAt.

Frontend contract:

Always call /api/trades/open to render the list.

When user clicks ×, POST to /api/trades/cancel.

Never expose ticket, broker, magic, or any MT5 internals in browser code.

Open‑trades API (GET /api/trades/open):

Query hedges where status = 'open' and return only { symbol, volume, openTime } formatted as ISO strings and human‑readable “lots.”

Cancel endpoint (POST /api/trades/cancel):

Accept { tradeOrderNumber }.

Lookup the hedge record for that internal ID, extract its ticket and broker.

Fire:

ts
Copy
await fetch('http://3.145.164.47/close_trade_by_magic', {
  method: 'POST',
  headers: {'Content-Type':'application/json'},
  body: JSON.stringify({
    magic: parseInt(hedge.ticket, 10),
    broker: hedge.broker,
    comment: 'User‑initiated cancel'
  })
});
On success update hedge.status = 'cancelled'; on error return 500.

Closed‑trades API (GET /api/trades/closed):

Query hedges where status != 'open' and return { symbol, volume, openTime, closedAt, status }.
